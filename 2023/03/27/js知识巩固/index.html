并发，并行，同步，异步

并发：单纯指的是计算机能同时执行多项任务，单线程和多线程的计算机都可以执行并发操作（单线程上是分配时间片段来实现并发，多线程则是通过并行来实现并发）

并行：将多个任务分配到多个处理器上同时执行任务，在同一时刻，有多个任务在执行。需要多核处理器来完成。所以单核处理器没有并行，只能有并发

同步和异步指的是两种不同的编程模式：

同步：上一个任务执行结束才能执行下一个任务，因此同步中没有并发和并行的概念

异步：在执行a的时候也可以同时执行b，典型的方式就是创建多线程编程，但是js是没有多线程的，但是可以做到单线程的并发，典型一是回调函数比如SetTimeout，函数创建之后会继续执行，当到达时间之后再执行回调函数

Object方法

  1. Object.getPrototypeOf(*)：返回参数对象的原型。这是获取原型对象的标准方法
  1. Object.setPrototypeOf(*，*)：参数对象设置原型，返回该参数对象。它接受两个参数，第一个是现有对象，第二个是原型对象
  1. Object.create()：以第一个参数为原型，返回一个实例对象，第二个参数为初始化的值，该值是实例对象的，不属于第一个参数。

返回的是一个对象，也可以是一个函数，但是不能是一个构造函数，指定了一个原型后，那么创建出来的新对象可以继承该原型上的属性和方法

  1. Object.prototype.isPrototypeOf(*) 是给实例用的，用于判断是否是参数的原型
  1. Object.is(a,b)：比较a和b是否相等，注意：+0和-0不相等，NaN和NaN相等
  1. Object.assign(target,b,c..)：对象的合并，将源对象（source）的所有可枚举属性，复制到目标对象
  1. Object.getOwnPropertyDescriptors(obj,key)：查看对象、对象属性的描述对象
  1. Object.protototype.hasOwnProperty(key)：指示对象是否具有指定的属性作为自身（不继承）属性,会忽略掉那些从原型链上继承到的属性
  1. Object.defineProperty(obj,prop,desc)：给目标对象的属性添加配置

正则表达式

  • 基本用法：

    /****/

  • 标志

    • 可以通过将它附加到正则表达式之后来使用它。 这里给出使用该标志的一个实例 /ignorecase/i ，这个字符串可以匹配字符串 ignorecase、igNoreCase 和 IgnoreCase
    • 若要多次搜寻或提取模式匹配，可以使用 g 标志
    • 通配符 . 将匹配任何一个字符
    • [aiu] 是只匹配字符 a、i 或者 u 的字符集
    • 要匹配小写字母 a 到 e，你可以使用 [a-e]
    • /[0-5]/ 匹配 0 和 5 之间的任意数字，包含 0 和5
    • 要创建否定字符集，需要在开始括号后面和不想匹配的字符前面放置脱字符（即^ ) ； /[^aeiou]/gi 匹配所有非元音字符。 注意，字符 .、!、[、@、/ 和空白字符等也会被匹配，该否定字符集仅排除元音字符
    • /a+/g 会在 abc中匹配到一个匹配项，并且返回 ["a"]。 因为 + 的存在，它也会在 aabc 中匹配到一个匹配项，然后返回 [“aa“],一次或多次
    • 匹配出现零次或多次的字符/*/，比如/go*/寻找g或者gooooo
    • /t[a-z]*i/ 应用于字符串 "titanic"。 这个正则表达式是一个以 t 开始，以 i 结束，并且中间有一些字母的匹配模式
    • 正则表达式 /t[a-z]*?i/ 匹配字符串 "titanic" 返回 ["ti"],它会匹配到满足正则表达式的字符串的最小可能部分
    • 除了在字符集中使用之外，插入符号（^）用于匹配文本是否在字符串的开始位置
    • 可以使用正则表达式的美元符号 $ 来搜寻字符串的结尾
    • \w 这个缩写等同于[A-Za-z0-9_]
    • \W 搜寻和 \w 相反的匹配模式。 注意，相反匹配模式使用大写字母。 此缩写与 [^A-Za-z0-9_] 是一样的
    • 查找数字字符的缩写是 \d，注意是小写的 d。 这等同于元字符 [0-9]，它查找 0 到 9 之间任意数字的单个字符；同样的，\D表示非数字
    • 使用 \s 搜寻空格，其中 s 是小写。 此匹配模式将匹配空格、回车符、制表符、换页符和换行符。 可以认为这类似于元字符 [ \r\t\f\n\v]；同样的，\S表示非空格
    • h{3，6}表示h要出现3-6次；{3，}表示至少出现三次；{3}表示出现三次
    • ? 指定可能存在的元素。 这将检查前面的零个或一个元素。 可以将此符号视为前面的元素是可选的；/colou?r/ 表示u是可选的，可有可无
    • 正向先行断言的用法是 (?=...)，其中 ... 就是需要存在但不会被匹配的部分；负向先行断言的用法是 (?!...)，其中 ... 是希望不存在的匹配模式

宏任务和微任务

  1. 宏任务（Web APIs）：大的任务，包括 新程序或子程序被直接执行（script）、事件的回调函数，setTImeout()、setInterval()、requestAnimationFrame、I/O操作、setImmediate、UI rendering等等，

    宏任务是浏览器规定的

  2. 微任务（micro task quene）：小的任务，包括promis.then().catch().finally()、MutationObserver、Object.observe、process.nextTick()，微任务是es6语法规定的

执行顺序：全局匿名函数：同步函数 — 微任务 —- 渲染 — 宏任务

浏览器加载过程

浏览器渲染引擎工作流程

  1. 第一步，构建 DOM 树：用 HTML Parser，分析 HTML 元素，构建一棵 DOM Tree；

  2. 第二步，生成样式表：用 CSS Parser，分析 CSS 文件和元素上的 inline 样式，生成页面的Style Rules；

  3. 第三步，构建 Render 树：将 DOM 树和样式表关联起来（Attachment），构建一棵 Render 树。每个 DOM 节点都有 attach 方法，接受样式信息，返回一个 render 对象（又名 renderer），这些 render 对象最终会被构建成一棵 Render Tree；

  4. 第四步，确定节点坐标：根据 Render 树结构，为每个 Render 树上的节点确定一个在显示屏上出现的精确坐标；

  5. 第五步，绘制页面：根据 Render 树和节点显示坐标，然后调用每个节点的 paint 方法，将它们绘制出来。creatDocumentFragment()，创建文档片段，将元素附加到文档片段里，然后将文档片段附加到dom树，该文档片段存在于内存，并不是在dom树上
